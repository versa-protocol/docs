# Bookings

A booking is a lightweight object you send to Versa when you may only have partial data about a transaction — like a hotel reservation — but not the final receipt. Versa evaluates the booking and, based on its contents, can retrieve the full receipt from the merchant, enrich the data, or forward it directly over the network.

**Typical flow for hotel bookings:**

1. Post a new booking with whatever data you have — property name, dates, amount, etc.
2. Versa evaluates the booking and, after checkout, kicks off receipt retrieval from the hotel.
3. When a receipt is ready, you can view it in your dashboard or receive a webhook.

## API Reference

Bookings are managed via API using your custodial client credentials.

### Create a booking

```
POST https://custodial.versa.org/booking
```

Provide as much data as possible — the more complete the booking, the better Versa can match and retrieve the receipt.

```bash
curl -X POST https://custodial.versa.org/booking \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "reference_id": "my_booking_123456789",
    "direct_account_id": "acct_123456789",
    "dryrun": false,
    "data": {
      ...
    }
  }'
```

#### Request Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `data` | `object` | Yes | Receipt data. Must validate against the [Versa schema](/receipt). |
| `reference_id` | `string` | No | A stable identifier for the object in your system (e.g. your booking ID). Use this to update or cancel the booking later. |
| `direct_account_id` | `string` | No | The direct account this booking belongs to. Defaults to the client's own direct account. |
| `dryrun` | `boolean` | No | If `true`, evaluates the booking without persisting it or triggering any actions. Defaults to `false`. |

### Update a booking

```
PUT https://custodial.versa.org/booking
```

You can update a booking while it is still pending. Identify the booking by its `id` or by the `reference_id` you provided at creation.

#### Example Request: Update a booking

```bash
curl -X PUT https://custodial.versa.org/booking \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "bkg_abc123def456",
    "reference_id": "my_booking_123456789",
    "data": {
      "schema_version": "2.2.0",
      "itemization": {
        "lodging": {
          "check_in": 1787315600,
          "check_out": 1787488400,
          ...
        }
        ...
      }
      ...
    }'
```

### Cancel a booking

```
DELETE https://custodial.versa.org/booking/{id}
```

You can cancel a booking while it is still pending. This may prevent a charge for receipt retrieval. Identify the booking by its `id` or by the `reference_id` you provided at creation.

#### Example Request: Cancel a booking by Id

```bash
curl -X DELETE https://custodial.versa.org/booking/bkg_abc123def456 \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET'
```

#### Example Request: Cancel a booking by Reference Id

```bash
curl -X DELETE https://custodial.versa.org/booking?reference_id=my_booking_123456789 \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET'
```
