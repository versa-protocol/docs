# Manifests

A manifest is a lightweight message you send to Versa when you have partial data about a transaction — like a hotel booking — but not the final receipt. Versa evaluates the manifest and, based on its contents, can retrieve the full receipt from the merchant, enrich the data, or forward it directly over the network.

**Typical flow for hotel bookings:**

1. Your system receives a hotel booking (e.g. from a card authorization or travel platform).
2. You create a manifest with whatever data you have — property name, dates, amount, etc.
3. Versa evaluates the manifest and kicks off receipt retrieval from the hotel.
4. When the receipt is ready, you receive a `manifest.action.completed` webhook.

## Quick Example: Hotel Booking

```bash
curl -X POST https://custodial.versa.org/manifest \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "event": "booking",
    "reference_id": "booking_123456789",
    "data": {
      "schema_version": "2.2.0",
      "header": {
        "invoice_number": "auth_1MzFN1K8F4fqH0lBmFq8CjbU",
        "currency": "usd",
        "total": 37000,
        "subtotal": 37000,
        "paid": 0,
        "invoiced_at": 1787288400,
        "location": {
          "address": {
            "tz": "America/New_York"
          }
        }
      },
      "itemization": {
        "general": null,
        "lodging": {
          "check_in": 1787115600,
          "check_out": 1787288400,
          "room": "0830",
          "location": {
            "name": "Courtland Grand Hotel",
            "address": {
              "street_address": "165 Courtland Street NE",
              "city": "Atlanta",
              "region": "GA",
              "country": "US",
              "postal_code": "30303"
            },
            "phone": "+14046596500"
          },
          "items": [
            {
              "description": "Room Charge",
              "amount": 18500,
              "date": "2026-08-21"
            },
            {
              "description": "Room Charge",
              "amount": 18500,
              "date": "2026-08-22"
            }
          ]
        }
      },
      "payments": [],
      "footer": {}
    }
  }'
```

This manifest tells Versa to retrieve the full folio from the Courtland Grand Hotel after the guest checks out on August 23. The API responds immediately with the manifest ID and a `pending` receipt retrieval action:

```json
{
  "id": "man_abc123def456",
  "state": "pending",
  "actions": [
    {
      "type": "receipt_retrieval",
      "status": "pending"
    }
  ]
}
```

## API Reference

Manifests are managed via API using your custodial client credentials.

### Create a Manifest

```
POST https://custodial.versa.org/manifest
```

Provide as much data as possible — the more complete the manifest, the better Versa can match and retrieve the receipt.

```bash
curl -X POST https://custodial.versa.org/manifest \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "event": "booking",
    "reference_id": "booking_123456789",
    "direct_account_id": "acct_123456789",
    "dryrun": false,
    "data": {
      ...
    }
  }'
```

#### Request Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | `string` | Yes | The event type (e.g. `booking`, `transaction`). |
| `data` | `object` | Yes | Receipt data. Must validate against the [Versa receipt schema](/schema). |
| `reference_id` | `string` | No | A stable identifier for the object in your system (e.g. your booking ID). Use this to update or cancel the manifest later. |
| `direct_account_id` | `string` | No | The direct account this manifest belongs to. Defaults to the client's own direct account. |
| `dryrun` | `boolean` | No | If `true`, evaluates the manifest without persisting it or triggering any actions. Defaults to `false`. |

#### Response

The response confirms the manifest was created and lists any actions Versa has triggered.

```json
{
  "id": "man_abc123def456",
  "state": "pending",
  "actions": [
    {
      "type": "receipt_retrieval",
      "status": "pending"
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier for the manifest (e.g. `man_abc123def456`). |
| `state` | `string` | Current state: `pending`, `completed`, or `canceled`. |
| `actions` | `Action[]` | Actions triggered by the manifest evaluation. |

**Action**

| Field | Type | Description |
|-------|------|-------------|
| `type` | `string` | The action type: `receipt_retrieval`, `receipt_forwarding`, etc. |
| `status` | `string` | Current status: `pending` or `completed`. |

### Update or Cancel a Manifest

```
PUT https://custodial.versa.org/manifest
```

You can update or cancel a manifest while it is still `pending`. This may prevent a charge for receipt retrieval. Identify the manifest by its `id` or by the `reference_id` you provided at creation.

```bash
curl -X PUT https://custodial.versa.org/manifest \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "man_abc123def456",
    "reference_id": "booking_123456789",
    "state": "canceled"
  }'
```

## Webhooks

Subscribe to the `manifest.action.completed` event to be notified when an action finishes. The webhook payload includes the manifest ID, the action type, and the results.
