# Manifests

Manifests are a flexible feature for partners to enrich and send data over the Versa network.

## Overview

Custodial clients may not always have a fully qualified receipt to send over the network.
A client might have a booking or a card transaction, but not the full itemization or finalized payment details.
In these cases, clients can create a **manifest**, which is a lightweight message containing the details they do have.
Based on the contents of the manifest, Versa may kick off a receipt retrieval flow, merchant enrichment, or send the data directly over the network if it is already in a valid state (for example, in the case of a pre-paid flight).

Manifests are exclusively managed via API using your custodial client credentials.

## Creating a Manifest

Specify as much data as possible to ensure the best possible outcome from the manifest evaluation.

```bash
curl -X POST https://custodial.versa.org/manifest \
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "data": {
      ...
    },
    "direct_account_id: "acct_123456789",
    "dryrun": false,
    "event": "booking",
    "handles": {
      "customer_email_domain": "acme.co"
    },
    "origin": {
      "id": "booking_123456789",
      "object": "booking",
    }
  }'
```

#### Request Parameters

| Field | Type | Description |
|-------|------|-------------|
| `data` | `object` | Data, must validate against the Versa receipt schema |
| `direct_account_id` | `nullable string` | The ID of the direct account this manifest is associated with. Defaults to the client's direct account. |
| `dryrun` | `nullable boolean` | If `true`, evaluates the manifest without persisting it or triggering any actions. Defaults to `false` |
| `event` | `string` | The event type that triggered this manifest (e.g. `booking`) |
| `handles` | `nullable TransactionHandles` | Optional transaction handles for routing any receipts |
| `origin` | `nullable OriginDescription` | Optional identifier of the source object in your system that generated this manifest; can be used for updating the manifest record. |

#### TransactionHandles

| Field | Type | Description |
|-------|------|-------------|
| `customer_email` | `nullable string` | The email address of the customer organization |
| `customer_email_domain` | `nullable string` | The email domain of the customer's organization |

#### OriginDescription

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | The authoritative identifier of the source object in your system |
| `object` | `nullable string` | The model or type of the source object in your system |


### Response
The response contains the manifest ID, its state, and any actions that were triggered as a result of the manifest creation. For example, if the manifest data was sufficient to trigger a receipt retrieval flow, the response would indicate that with a pending `receipt_retrieval` action.

### Example Response

```json
{
  "id": "man_abc123def456",
  "state": "pending",
  "actions": [
    {
      "type": "receipt_retrieval",
      "status": "pending"
    }
  ]
}
```

#### Response

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | The unique identifier for the created manifest |
| `state` | `string` | The current state of the manifest (e.g. `pending`, `completed`) |
| `actions` | `Action[]` | List of actions triggered by the manifest evaluation |

#### Action

| Field | Type | Description |
|-------|------|-------------|
| `type` | `string` | The type of action (e.g. `receipt_retrieval`, `receipt_forwarding`) |
| `status` | `string` | The current status of the action (e.g. `pending`, `completed`) |

## Updating or Canceling a Manifest

You may update or cancel manifests while they are in a pending state, which may prevent a charge for receipt retrieval.
If you provided an `origin` when creating the manifest, you can use that to identify the manifest to update or cancel; otherwise, you can use the manifest ID returned in the creation response.

```bash
curl -X PUT https://custodial.versa.org/manifest
  -H 'Authorization: Basic CLIENT_ID:CLIENT_SECRET' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "man_abc123def456",
    "state": "canceled",
    "origin": {
      "id": "booking_123456789",
      "object": "booking",
    }
  }'
```

## Webhooks

Subscribe to the `manifest.action.completed` webhook to be notified when actions triggered by the manifest evaluation are completed. The webhook payload will include the manifest ID, the action type, and the results of the action.
